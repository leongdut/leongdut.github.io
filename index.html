Node.js-Express实现文件上传

2018-05-09

基本知识补充

Express 框架核心特性：

  • 可以设置中间件来响应HTTP请求使得前后端开发者各自高效分工
  • 通过定义路由表来执行不同的HTTP请求动作
  • 通过向模板传递参数来动态喧嚷HTML页面
    请求和响应

回调函数

通过request和response对象来处理请求和响应数据：

app.get('/', function (req, res) {// --})
  • Request 对象 - 表示 HTTP 请求，包含了请求查询字符串，参数，内容，HTTP 头部等属性。
  • Response 对象 - 象表示 HTTP 响应，即在接收到请求时向客户端发送的 HTTP 响应数据。

文件上传

文件上传总计分为三种方式：

  • 通过flash,activeX等第三方插件实现文件上传功能。
  • 通过html的form标签实现文件上传功能，优点：浏览器兼容好。
  • 通过xhr level2的异步请求，可以搜一下formData对象。
    这一次我按照学校实验课要求实现第二点的文件上传方式。

设计模式-Facade模式

2018-05-07

通俗解释：为子系统的一组接口提供一个统一接口。也就是说定义一个更高层的接口，使得子系统更加容易使用。

关键特征

意图：希望简化原有系统的使用方式。需要定义自己的接口。
问题：只需要某个复杂系统的字迹，或者，需要以一种特殊的方式与系统交互。
解决方案：Facade为原有的系统的客户提供了一个新的接口。
参与者与协调者：为客户提供一个简化的接口，使得系统更容易使用。
效果：Facade模式简化了子系统的使用过程。同时，Facade并不完整，客户可能不能使用到某些功能。
实现方式：1.定义一个或多个新类，具备所需要的接口；2.让新的类使用原有的系统。

应用场景


1.如果Facade可以设计成没有存储状态的，那么一个Facade对象就能够被多个其他对象使用。
2.假设除了使用系统中原有的功能以外，还需要一些新的功能——比如，记录对特定历程的所有调用。在这种强开下，我们就可以为Facade类所写的方法里可以为新的功能添加一些新的例程。这仍然满足Facade模式。
3.最基本的，不仅可以用来通过方法调用创建更简单的接口，还可以用来减少客户必须处理的对象数量。
